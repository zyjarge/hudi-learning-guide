## 阶段一：理论学习

### 目标

- 理解本章核心概念
- 了解 Hudi 的设计原理
- 解答所有疑问

### 流程

1. **知识点讲解**
   - 讲师（AI）提供本章知识点
   - 使用通俗易懂的语言
   - 配合图表和示例

2. **外部资料推荐**
   - 官方文档
   - 技术博客
   - 视频教程
   - 社区讨论

3. **问答环节**
   - 学员提出问题
   - 讲师逐一解答
   - 直到学员确认理解

### 交付物

- [ ] 知识点笔记
- [ ] 所有问题已解答
- [ ] 学员确认理解

## 阶段二：动手实操

### 目标

- 实践本章知识
- 编写可运行的代码
- 理解运行结果

### 流程

1. **编写示例代码**
   - 基于本章知识点
   - 覆盖核心操作

2. **运行验证**
   - 执行代码
   - 检查输出

3. **结果分析**
   - 解释输出含义
   - 理解数据流

### 交付物

- [ ] 代码脚本已创建
- [ ] 代码执行成功
- [ ] 运行结果已分析

## 阶段三：章节测验

### 目标

- 验证理论理解
- 确认实操能力

### 测验规则

| 项目 | 要求 |
|------|------|
| 题目数量 | 5-10 道 |
| 题型 | 选择题 + 简答题 |
| 及格线 | 60% 正确率 |
| 补考 | 允许，需重新学习 |

### 测验形式

**选择题示例：**
```
Q: Hudi 的全称是什么？
A) Hadoop Unified Data Infrastructure
B) Hadoop Upsert Deletion and Incremental
C) High Availability Data Infrastructure
D) Hybrid Unified Data Interface
```

**简答题示例：**
```
Q: COW 和 MOR 的主要区别是什么？
A: (需要用自己的话描述)
```

### 通过标准

- 选择题正确率 ≥ 60%
- 简答题能清晰表达核心概念

### 未通过处理

1. 回顾理论部分
2. 重新运行示例代码
3. 重新参加测验
