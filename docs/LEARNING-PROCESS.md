本文档规定了 Apache Hudi 学习课程的标准化学习流程。每章学习必须严格按照此流程执行。

## 阶段一：理论学习

### 目标

- 理解本章核心概念
- 了解 Hudi 的设计原理
- 解答所有疑问

### 流程

1. **知识点讲解**
   - 讲师（AI）提供本章知识点
   - 使用通俗易懂的语言
   - 配合图表和示例

2. **外部资料推荐**
   - 官方文档
   - 技术博客
   - 视频教程
   - 社区讨论

3. **问答环节**
   - 学员提出问题
   - 讲师逐一解答
   - 直到学员确认理解

### 交付物

- [ ] 知识点笔记
- [ ] 所有问题已解答
- [ ] 学员确认理解

## 阶段二：动手实操（向导式教学）

### 目标

- 实践本章知识
- 理解每个步骤的作用
- 独立完成操作

### 教学方式：向导式（菜谱式）

**讲师（AI）将采用以下方式引导：**

```
┌─────────────────────────────────────────────────────────────┐
│                    向导式实操流程                             │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  第 0 步：实验前置说明                                        │
│  ────────────────────                                        │
│  • 列出本次实验涉及的知识点（1, 2, 3...）                     │
│  • 说明实验目标                                              │
│                                                              │
│  第 1 步：                                                    │
│  ───────────                                                 │
│  • 给出具体指令（如：打开终端、输入命令）                      │
│  • 说明这一步的目的                                          │
│  • 等待学员执行并反馈                                         │
│                                                              │
│  第 2 步：                                                    │
│  ───────────                                                 │
│  • 给出下一步指令                                             │
│  • 解释代码/命令的含义                                        │
│  • 等待学员执行并反馈                                         │
│                                                              │
│  ...                                                          │
│                                                              │
│  第 N 步：结果验证                                            │
│  ─────────────                                               │
│  • 确认输出符合预期                                           │
│  • 分析关键结果                                               │
│  • 总结收获                                                   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 交互规则

| 场景 | 讲师行为 |
|------|---------|
| 学员提出问题 | 暂停步骤，解答问题后继续 |
| 学员执行出错 | 指出错误原因，给出正确做法 |
| 学员理解困难 | 换一种方式解释，或提供更多背景 |
| 学员超时未回复 | 询问进度，确认是否需要帮助 |

### 向导式示例

**❌ 错误方式（直接给代码）：**
```
请运行以下代码创建 Hudi 表：
spark.read.format("hudi").options(**options).save(path)
```

**✅ 正确方式（分步引导）：**
```
【实验目标】创建第一个 Hudi 表并读取数据

【知识点】
1. Hudi 表需要配置 recordkey.field（主键）
2. Hudi 表需要配置 precombine.field（排序字段）
3. write.format("hudi") 指定使用 Hudi 格式

【第 1 步】编写创建表的代码
请在脚本中添加以下内容：
...
请执行并告诉我结果。
```

### 交付物

- [ ] 每一步骤已执行
- [ ] 每一步骤结果已确认
- [ ] 学员能解释操作的含义

## 阶段三：章节测验

### 目标

- 验证理论理解
- 确认实操能力

### 测验规则

| 项目 | 要求 |
|------|------|
| 题目数量 | 5-10 道 |
| 题型 | 选择题 + 简答题 |
| 及格线 | 60% 正确率 |
| 补考 | 允许，需重新学习 |

### 测验形式

**选择题示例：**
```
Q: Hudi 的全称是什么？
A) Hadoop Unified Data Infrastructure
B) Hadoop Upsert Deletion and Incremental
C) High Availability Data Infrastructure
D) Hybrid Unified Data Interface
```

**简答题示例：**
```
Q: COW 和 MOR 的主要区别是什么？
A: (需要用自己的话描述)
```

### 通过标准

- 选择题正确率 ≥ 60%
- 简答题能清晰表达核心概念

### 未通过处理

1. 回顾理论部分
2. 重新运行示例代码
3. 重新参加测验
